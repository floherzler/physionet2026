# Guild Charter

Updated on: `2026-03-04`

Team norms for collaboration quality, speed, and decision hygiene.

## Operating Principles

- Keep changes small and reviewable.
- Prefer explicit tradeoffs over implicit assumptions.
- Capture key decisions in writing where others can find them.

## Working Agreement

- Quest-first execution for implementation work.
- Validation evidence before marking work complete.
- No silent scope expansion after acceptance criteria are set.

## Decision Protocol

- Record decision: what was chosen and why.
- Record alternatives considered.
- Record rollback trigger (what would make us reverse course).

## Fill this in

### Review Expectations

- Current mode: fast iteration with flexible ownership by code area.
- Merge gates: intentionally lightweight for now; tighten after baseline integration stabilizes.
- Reviewer/check policy: to be defined when collaboration load increases.

### Communication Defaults

- Non-trivial implementation decisions should be captured in quest docs and linked PR notes.
- Use synchronous discussion when blockers span multiple modalities or affect core challenge contracts.

## Questions to answer

- At what point do we introduce stricter merge gates (e.g., after first stable end-to-end run)?
- Do we want explicit ownership by component once the team workload increases?
